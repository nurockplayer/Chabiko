## Goal

Build the learner-facing pages for lessons, vocabulary, and phrasebook content.

## Scope

- Use the existing content model to render lesson pages.
- Render vocabulary and phrasebook browsing surfaces.
- Ensure mobile and desktop readability.
- Use `design-taste-frontend` before detailed page design.

## Out of scope

- Practice engine.
- User accounts.
- Audio library.

## Acceptance criteria

- [ ] Learner can browse lesson, vocabulary, and phrasebook pages.
- [ ] Pages render from structured content files.
- [ ] Mobile and desktop layouts do not overlap text or controls.
- [ ] The first screen exposes useful learning content, not only marketing copy.
